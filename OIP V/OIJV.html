Object Oriented Programming in Python - V

Object Oriented Programming (OOP) in Python allows programs to organize data and functionality using classes and objects. It is an important programming paradigm that helps in creating reusable, modular, and maintainable code by combining data and methods into a single unit called an object. OOP improves software development by enabling concepts such as encapsulation, inheritance, polymorphism, and abstraction. These concepts help reduce code duplication, increase security, simplify maintenance, and make applications easier to expand and manage. Object Oriented Programming helps in representing real-world entities as objects, making complex systems easier to design and understand. It also supports better organization of programs by dividing functionality into independent classes. In this lab, you will study different Object Oriented Programming concepts in Python, including creating classes and objects, using constructors, implementing encapsulation for data hiding, applying inheritance for code reusability, understanding polymorphism for flexible behavior, and using abstraction to hide implementation details while exposing only essential functionality.

Object Oriented Programming (OOP)

Object Oriented Programming (OOP) is a programming approach that organizes programs using Classes and Objects. OOP helps in creating reusable, secure, maintainable and scalable applications.

Class and Object

A Class is a blueprint used to create objects. An Object is the actual entity created from the class. 

class Student:

    def show(self):
        print("Hello Student")

obj = Student()

obj.show()

Constructor (__init__ Method)

Constructor automatically executes when an object is created. Used to initialize object variables. 

class Student:

    def __init__(self,name):

        self.name=name

obj=Student("Sachin")

print(obj.name)

1. Encapsulation

Encapsulation means binding data and methods together inside one class and restricting direct access to data. Purpose: • Data Hiding • Security • Controlled Access Public Variables: Accessible anywhere Private Variables: Accessible only inside class (using double underscore) 

class Account:

    def __init__(self):

        self.__balance=1000

    def show(self):

        return self.__balance


obj=Account()

print(obj.show())

2. Inheritance

Inheritance allows one class to acquire properties and methods from another class. Parent Class: Original class Child Class: Derived class Benefits: • Reusability • Reduced Code 

class Parent:

    def display(self):

        print("Parent Method")


class Child(Parent):

    pass


obj=Child()

obj.display()

3. Polymorphism

Polymorphism means one method behaves differently for different objects. Poly = Many Morph = Forms Benefits: • Flexibility • Easy Extension 

class Dog:

    def sound(self):

        print("Bark")


class Cat:

    def sound(self):

        print("Meow")


a=Dog()

b=Cat()

a.sound()

b.sound()

Method Overriding

Method overriding happens when child class creates its own version of parent method. 

class Parent:

    def show(self):

        print("Parent")


class Child(Parent):

    def show(self):

        print("Child")


obj=Child()

obj.show()

4. Abstraction

Abstraction means showing only essential information and hiding implementation details. Abstract Class: Cannot create object directly. Abstract Method: Method without implementation. 

from abc import ABC
from abc import abstractmethod


class Vehicle(ABC):

    @abstractmethod

    def start(self):

        pass


class Car(Vehicle):

    def start(self):

        print("Car Started")


obj=Car()

obj.start()

Difference Between Encapsulation and Abstraction

Encapsulation
→ Hides Data
