Exception Handling in Python - VI

Exception Handling in Python allows programs to manage errors and exceptional conditions gracefully. It is an important programming paradigm that helps in creating robust and reliable code by providing mechanisms to handle runtime errors. Exception Handling improves software development by enabling developers to anticipate and manage potential errors, ensuring that the program can continue to run even when unexpected situations occur. This leads to better user experience and more maintainable code. In this lab, you will study different Exception Handling concepts in Python, including creating try-except blocks, raising exceptions, handling multiple exceptions, and using finally blocks for cleanup operations.

Exception Handling in Python

Below are Exception Handling based programming questions with implementation.

Exception Handling

Exception handling deals with runtime errors and provides a way to respond to exceptional conditions without crashing the program. In Python this is done with try, except, else and finally blocks. 

# Example: handling division by zero
try:
  a = 10
  b = 0
  result = a / b
except ZeroDivisionError as e:
  print("Error:", e)
  result = None
else:
  print("Result is", result)
finally:
  print("Cleanup if needed")

Exception Handling

Exception Handling is used to handle runtime errors and prevent the program from crashing. Purpose: • Avoid Program Termination • Handle Errors Gracefully • Improve Program Reliability 

try:
    a = 10
    b = 0

    print(a/b)

except:
    print("Error Occurred")

Try and Except

The try block contains risky code. The except block executes when an error occurs. 

try:
    num = int(input("Enter Number: "))

except:
    print("Invalid Input")

Handling Specific Exception

Specific exceptions allow handling particular types of errors. 

try:
    a = 10
    b = 0

    print(a/b)

except ZeroDivisionError:
    print("Cannot divide by zero")

Multiple Exceptions

One try block can handle multiple exceptions. 

try:
    num = int(input())

    print(10/num)

except ValueError:
    print("Invalid Number")

except ZeroDivisionError:
    print("Division by Zero")

Else Block

Else block executes only if no exception occurs. 

try:
    a = 10
    b = 2

    print(a/b)

except:
    print("Error")

else:
    print("Executed Successfully")

Finally Block

Finally block always executes whether exception occurs or not. Used for: • Closing Files • Database Connections • Cleanup Operations 

try:
    print(10/2)

except:
    print("Error")

finally:
    print("Program Finished")

Raise Exception

raise keyword is used to generate custom exceptions. 

age = -5

if age < 0:
    raise Exception("Age cannot be negative")

User Defined Exception

Users can create their own exception classes. 

class InvalidAge(Exception):
    pass

age = -1

if age < 0:
    raise InvalidAge("Invalid Age")
